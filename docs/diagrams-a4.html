CLASS DIAGRAM & DFD

The .NET 8 backend domain expressed as a UML class diagram, and three levels of data-flow — context, sub-processes, and the spatial-audit decomposition.

4.1 · مخطط الفئات (UML) Class diagram (.NET 8 backend)
Citizen, Officer, DepartmentManager, Property, RegistrationRequest, PropertyNft, OwnershipHistory, BlockchainService, SmartContract, IpfsService, DigitalIdCard, NfcCardSecret, SsiWallet, SsiCredential, JwtTokenService, StorageService, PadesSigner.
4.2 · تدفق البيانات · المستوى ٠ DFD · Level 0 (context)
Sarh Platform as a single process with all external entities (Citizen, Registry officer, Department manager, SSI, Map, Notification, NFT minting service).
4.3 · تدفق البيانات · المستوى ١ DFD · Level 1
Five main processes (Account & Identity, Land Claim Processing, Spatial Auditing, NFT Documentation & Minting, Notifications) with data stores including D3 Blockchain Ledger.
4.4 · تدفق البيانات · المستوى ٢ (التدقيق المكاني) DFD · Level 2 (spatial)
Decomposition of process 3.0 — Document Review → Spatial Conflict Check → Technical Map Generation → Status Update.
4.5 · تدفق النظام الشامل System flow (end-to-end)
citizen → officer → manager → ID issuer → admin, with the .NET 8 API, SQL Server, ACA-Py, IPFS, and the blockchain at the centre.
05
مخططات التسلسل
SEQUENCE DIAGRAMS

Step-by-step walk-throughs for sign-in, property approval (split into submit / mint / verify so each fits on A4), and ID-card issuance.

5.1 · تسجيل الدخول Sign-in
POST /api/v1/auth/sign-in from browser → proxy → AuthController → BCrypt.Verify → JWT issuance.
5.2 · موافقة العقار (١ / ٣) · التقديم والمراجعة والاعتماد Property approval (1/3) · Submit + Review + Manager approval
Citizen submission → Registry-officer recommendation → Department-manager final approval (with decree number).
5.3 · موافقة العقار (٢ / ٣) · السند الإلكتروني وسك NFT Property approval (2/3) · Deed PDF + SSI + IPFS + NFT mint
PAdES-signed deed → SSI VC issuance → IPFS pin → ERC-721 mint on-chain.
5.4 · موافقة العقار (٣ / ٣) · التسليم والتحقق العام Property approval (3/3) · Delivery + public verify
Citizen download (signed PDF + QR) → public verifier scans → API checks PDF hash + on-chain ownerOf.
5.5 · موافقة العقار · مرجع شامل Property approval · full reference
Combined sequence including all 12 participants and 6 phases. Dense — split versions (5.2 – 5.4) are easier to read.
5.6 · إصدار البطاقة الرقمية ID-card issuance (5-step wizard)
Wizard → citizen create → card issue → NFC encode → SSI VC → print.

رحلات المستخدم
USER JOURNEYS

The two end-to-end journeys built into the platform — citizen registering a property and receiving an NFT licence, and officer/manager approving submissions and triggering minting.

6.1 · رحلة المواطن Citizen journey
Landing → sign in → register property → wait for officer + manager → receive deed + NFT licence. Includes "My Wallet (NFT licences)" branch.
6.2 · رحلة الموظف ومدير القسم Officer + Department-manager journey
Officer queue → review → recommend; manager approval queue → final approve → triggers PAdES + SSI + IPFS + mint chain.
